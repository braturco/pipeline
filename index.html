Power Automate setup guide
01
Upload the exported Excel to a SharePoint document library, e.g. Pipeline Reviews / 2025-Q2.xlsx. Format the data as a Table (Insert → Table in Excel) before uploading so PA can use List rows present in a table.
02
Create a Power Automate flow. Trigger: Manually trigger a flow (or schedule). Action: List rows present in a table using the Excel / SharePoint connector pointing to your uploaded file.
03
Add a Select action to extract unique emails from the rows. Then Apply to each unique email — filter the full row list to only that owner's rows using Filter array where Owner Email equals the current email.
04
Build an HTML table string for each owner. Use a Create HTML table action or compose the HTML manually with the columns: Name, Account, Net CV, Probability, Close Date, Est Start, Est End, Stale Flag. Stale rows can be highlighted with inline style="background:#fff3cd".
05
Send via Send an email (V2). To: current email. Subject: Pipeline review request — [count] opportunities assigned to you. Body: HTML table + a button/link to a Microsoft Form or Power Apps form. The form should capture: Opportunity Code, Is Active (Yes/No/Uncertain), Revised Close Date, Revised Start Date, Revised End Date, Notes.
06
Form responses write to a SharePoint list. When you re-import here next cycle, responses can be loaded via the JSON state restore to carry forward any manual updates you applied.
07
On re-import: export fresh data from Power BI, import here. The tool merges on Opportunity Code — your review statuses, notes, and any manual email overrides are preserved. New opportunities are appended automatically.
